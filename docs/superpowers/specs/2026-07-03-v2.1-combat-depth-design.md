# v2.1 Combat Depth — Design (M10)

**Status:** design / self-approved on autopilot (user delegated: "carry out all of those improvements on your own and deploy … remember proper workflow"). User veto window open before build starts.

**Milestone:** M10 (combat decision redesign). Follows M9 (roster→40, roster-only). Precedes M11 (fighter avatars).

## 1. Problem / Why

The live v2 round decision is `RoundIntent { where: 'strike'|'wrestle'|'grapple', target: 'head'|'body', approach: 'pressure'|'technical'|'counter' }`. Two real design flaws (user-reported):

1. **Striking vocabulary bolted onto grappling.** `approach` (Pressure / Technical / Counter) is boxing language, but it is applied identically to `wrestle` and `grapple`. "Counter a pressure" while shooting a double-leg is nonsense. The secondary decision must fit the action.
2. **No wrestle→ground sequencing.** `wrestle` and `grapple` are flat sibling choices. Real MMA: you wrestle to GET the fight to the mat, THEN decide ground-and-pound vs submission. The model has no such "then."

The 9-stat sheet already supports grappling (`takedowns/takedownDef`, `submissions/submissionDef`) and `finish.ts` already has a submission finish path. So this is a **decision-layer redesign**, not an engine rewrite. The exchange math, stamina economy, judges, and the finish-window machinery are reused.

## 2. Goal

Each round's decision reflects the chosen action's context, and wrestling has a real two-stage payoff: **shoot → (if it lands) choose Ground & Pound or Submission**. Keep it bounded — no persistent multi-round ground position.

## 3. New decision model (locked)

Replace the flat record with a discriminated-union `RoundIntent`:

```ts
export type StrikeTactic = 'pressure' | 'counter' | 'pickApart';
export type GroundPlan   = 'ground-and-pound' | 'submission';

export type RoundIntent =
  | { kind: 'strike'; target: 'head' | 'body'; tactic: StrikeTactic }
  | { kind: 'wrestle' };   // the ground plan is chosen later, in the ground window
```

### Strike branch (stand-up) — math preserved
- **Pressure** = aggressive (high attack / low defense), **Counter** = reactive (low attack / high defense, punishes an opponent's Pressure), **Pick-apart** = technical/measured (neutral).
- These map 1:1 onto the EXISTING multipliers: `pressure→pressure`, `counter→counter`, `pickApart→technical`. Striking exchange, striking finish (rocked head → KO), and the counter-beats-pressure read are all preserved.

### Wrestle branch (take it down) — the new depth
A takedown attempt resolved by `takedowns` vs `takedownDef` (+ Fight-IQ tilt + seeded swing + stamina effort):
- **Takedown lands** → fight enters a **ground window** (`phase: 'ground-window'`); the attacker is in top control and picks a ground action (same UX pattern as the finish-window pressure decision):
  - **Ground & Pound** → head damage from top. Ground-striking rating = a blend of `striking` and `takedowns` (a strong wrestler with weak hands can still hurt you), measured against the opponent's `strikingDef`, floored so it's never inert. If it rocks the opponent (crosses `ROCKED_HEAD_DMG(chin)`) → **KO finish** (reported as `method:'KO'`, i.e. a TKO). Else the round advances (back to standing next round — no persistent position).
  - **Submission** → a submission finish attempt (`submissions` vs `submissionDef`, commit/measure roll like `finishStep`). Success → **submission win**; failure → opponent escapes, energy cost, round advances.
- **Takedown stuffed** → the aggressor is exposed: opponent lands a counter (opponent scores the round + player takes head damage), round advances. This risk is what makes careless shooting punishable (feeds balance BAND 2).

### Remove the flat `grapple` top-level
Grappling now happens ONLY through wrestle→ground. The `grappler` archetype and `submissions/submissionDef` stats remain meaningful: grapplers wrestle then submit, and submission defense still matters when someone takes YOU down. `Where`/`PHASE_OFFENSE`/`PHASE_DEFENSE` and the `finish.ts` grapple-submission read are updated accordingly.

## 4. Opponent AI (`opponentIntent`)

Returns the new union.
- **Strike vs wrestle:** compare the opponent's best striking edge (`striking − player.strikingDef`) against its best grappling edge (`takedowns − player.takedownDef`, considering `submissions − player.submissionDef`). Archetype-driven in practice: strikers/brawlers strike, wrestlers/grapplers wrestle, allrounders pick by matchup. `fightNumber` still nudges aggression.
- **Ground action (when the opponent's takedown lands):** pick by the player's weaker defense — low `submissionDef` → Submission, else Ground & Pound. Mirrors the player.
- Strike tactic chosen as today (aggression scales with `fightNumber`) but only inside the strike branch.
- RNG draws stay uniform/upfront (preserve `opponent.test.ts` determinism + uniformity invariants).

## 5. Engine surface changes

- `intents.ts`: new union + type guards + labels; remove `Where/Approach/APPROACHES` (or narrow to strike tactics). Provide helpers the UI/AI need (e.g. `STRIKE_TACTICS`, `GROUND_PLANS`, `intentPhase(intent)`).
- `fightState.ts`: `FightPhase` gains `'ground-window'`; add a `GroundWindow { side; stepsLeft }` (or reuse `window` with a `kind` discriminator). `opponentIntent` rewritten. `RoundLogEntry` still records both intents.
- `resolve.ts`: branch on `intent.kind`. Strike → existing exchange. Wrestle → takedown contest → open ground-window on success / counter on stuff. New tuning constants for takedown threshold + GnP.
- `finish.ts`: add `groundStep(state, choice: GroundPlan): FightState` (mirrors `finishStep`); GnP damage + rocked→KO; submission finish roll. Keep `finishStep` for standing KO/submission reads.
- `judges.ts`, `stamina.ts`, `run.ts`: unchanged except `staminaCost` signature (now keyed on intent kind/tactic instead of where/approach).

## 6. Persistence (`runStorageV2`)

Add the `'ground-window'` phase + payload invariant to `isValidFightState` (mirror the finish-window contract exactly so engine output is never false-rejected): `ground-window ⇒ window non-null (ground kind) && outcome===null`. Deterministic resume must handle a persisted ground-window (drive it with `groundStep`). Existing phase↔payload and phase↔fighter invariants retained.

## 7. Balance / tuning (re-tune — this is what M9 deferred)

Decision space changed, so the harness's play policies are redefined:
- **Good play:** pick-apart when even; pressure when opponent hurt/gassed; counter when opponent pressures; wrestle when the opponent's takedownDef/submissionDef is the weak point; in the ground window, submission vs low submissionDef else ground-and-pound.
- **Careless play:** fixed dumb policy (e.g. always Pressure, or always shoot regardless of matchup).

Re-measure over 300 seeds and tune constants to the **strengthened** bands M9 deferred (never weaken below the current live floor; Achievable-floor rule: if a target is unreachable without pushing another band red, back THAT ONE assertion to best-stable-achieved but still ≥ its original live floor):
- **BAND 1** finish rate (good, avg fights 1–10) ≥ **0.30** (live floor 0.25).
- **BAND 2** careless@1 ≤ **0.72** (live 0.80) AND good@1 − careless@1 ≥ **0.20** (live 0.15).
- **BAND 3** good@9 ≥ **0.45** and good@10 ≥ **0.45** (live 0.40; never 0).
- **BAND 4** no runaway (late win-rate < early, and < 0.90). Unchanged.
Keep `opponent.test.ts` invariants (monotonic target curve, ≤90, span ≤40, ±2 avg) green.

## 8. UI

- **IntentPanelV2** → primary **Strike vs Wrestle**; Strike reveals **Head/Body** + three tactics (Pressure / Counter / Pick-apart); Wrestle is a single commit ("Shoot for the takedown"). No striking words on the wrestle path.
- **New ground-window panel** (reuse the finish/pressure-decision panel styling): "Takedown landed — top control" → **Ground & Pound** / **Submission**.
- **FightView** controller handles `phase==='ground-window'` → renders the ground panel → calls `groundStep`; existing finish-window handling unchanged.
- Display helpers/labels updated so no grappling decision shows striking vocabulary.

## 9. Out of scope (YAGNI)

Persistent multi-round ground position, scrambles, get-up mechanics, clinch, new stats, new archetypes, specific submission types (guillotine/RNC), striking on the feet sub-positions. Each takedown resolves within its round. Fighter avatars are a separate milestone (M11).

## 10. Testing (strict TDD)

Domain: new intents union + guards; `resolveRound` strike branch reproduces current results (regression), wrestle-lands→ground-window, wrestle-stuffed→counter+damage; `groundStep` GnP (damage, rock→KO) and submission (win/fail→escape); `opponentIntent` style-appropriate selection + RNG uniformity; persistence round-trips a real engine-produced ground-window and rejects a corrupt one; balance bands green + deterministic; opponent invariants green; full-run integration across a complete title run. UI: IntentPanelV2 strike/wrestle rendering, ground panel, FightView controller wiring, deterministic mid-fight resume from a persisted ground-window.
